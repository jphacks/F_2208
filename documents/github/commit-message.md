# コミットメッセージ フォーマット

```
1行目: 変更内容の要約（タイトル, 概要を簡潔に書く）
2行目: 改行（3行目がなければ必要なし）
3行目以降: 変更した内容・理由・詳細（任意）
```

## 1 行目

```
[コミット種別]: 要約 #[Issue番号]
```

ex.

```
add: Install Material UI #5
```

### コミット種別

上 3 つは最低限扱えるようにする。

- **add: 新規追加（機能, ファイル）**
- **fix: バグ修正**
- **remote: 削除**

---

- change: 仕様変更
- hotfix: クリティカルなバグ修正
- update: 機能修正（バグではなもの）
- clean: 整理（リファクタリングなど）
- revert: 変更の取り消し

### 要約

日本語でも OK
